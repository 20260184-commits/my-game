# Week 3 실습 기록

## 목표
Pygame을 활용한 기본 게임 환경 구축, 사용자 입력 처리, 그리고 시간 기반의 객체 생성 상호작용 구현

## AI 대화 기록

**Q1 : Pygame의 기본 게임 루프와 작동 원리는?
- AI 답변 : while 루프 내에서 이벤트 처리 -> 화면 초기화 -> 그리기 -> 업데이트 순서 설명
- 적용 결과: Pygame의 전체적인 흐름을 이해하고 기초 프레임 완성

**Q2 : FPS 표시와 속도 조절은 어떻게 하나요?
- AI 답변 : clock.tick(60)으로 프레임 제한, font.render로 텍스트를 이미지로 변환하여 출력
- 적용 결과: 게임 속도 안정화 및 실시간 성능 모니터링 기능 추가

**Q3 : 방향키 입력에 따른 캐릭터 이동과 상태 변화는?
- AI 답변: pygame.key.get_pressed()와 any()를 활용하여 입력 감지 및 상태 변수(색상, 위치) 업데이트
- 적용 결과: 이동 시 색상이 변하고 입력이 없을 때 원위치로 돌아오는 상호작용 구현

**Q4 시간 제한이 있는 랜덤 객체 생성 로직은?
- AI 답변: pygame.time.get_ticks()를 사용하여 생성 시간을 기록하고, 현재 시간과의 차이를 계산하여 객체 생성 및 삭제(3초 유지)
- 적용 결과: 2초마다 랜덤 위치에 검은색/흰색 사각형이 생성되고 3초 후 사라지는 기능 완성

## 어려웠던 점
- pygame.key.get_pressed()와 pygame.KEYDOWN 이벤트의 차이점 구분 (연속 입력 vs 단발 입력)
- 여러 개의 사각형 객체를 리스트로 관리하고, 시간 조건에 따라 적절하게 필터링하여 삭제하는 로직 설계
- 중심 좌표를 기준으로 도형(원, 사각형, 삼각형)의 위치를 정렬하는 좌표 계산

## 배운 점
- any() 함수를 사용하면 여러 개의 키 입력을 매우 간결하게 조건문으로 처리할 수 있음
- pygame.time.get_ticks()는 게임 내에서 시간 기반의 이벤트를 제어하는 핵심 도구임
- 객체의 속성(위치, 색상 등)을 딕셔너리에 담아 리스트로 관리하면, 다수의 객체를 생성/삭제할 때 매우 효율적임
