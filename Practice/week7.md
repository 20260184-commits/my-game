## 사용한 AI 프롬프트 (요약)
1. **UI 개선 요청:** "UI 글씨랑 크기가 말끔하지 않으니 수정해줘. UI만 손보고, Base64 이미지는 내가 넣을 테니 너는 넣지 말고 전체 코드를 작성해줘."
2. **로직 복구 요청:** "너무 간소화한 것 같아. (보스 페이즈, 충돌 판정 등) 기존 게임 로직에 문제가 없도록 로직을 유지해서 다시 작성해줘."
3. **배경 음악 추가:** "게임 배경음악(BGM)을 반복 재생되도록 추가하고 싶어."
## AI 답변에서 도움이 된 것
- **`draw_text` 헬퍼 함수:** 텍스트에 그림자 효과를 주고 위치를 중앙 정렬하는 함수를 제공받아 UI 가독성을 매우 쉽게 높일 수 있었습니다.
- **코드 구조화:** UI 변경 작업 시 핵심 게임 로직(보스 상태 머신, 충돌 감지)이 섞이지 않도록 모듈화하여 관리하는 방법을 배웠습니다.
- **`pygame.mixer` 활용:** `pygame.mixer.music.play(-1)`을 통해 별도의 복잡한 구현 없이 간단하게 BGM을 무한 반복시키는 방법을 학습했습니다.

## AI 답변을 수정하거나 버린 
- **AI의 과도한 간소화 코드:** 첫 번째 UI 개선 답변에서 AI가 가독성을 위해 기존의 복잡한 보스 로직을 너무 많이 삭제했습니다. 이를 그대로 사용하면 게임이 작동하지 않으므로, 기존 로직을 모두 보존하도록 다시 요청하여 수정했습니다.
- **Base64 데이터:** AI가 임의로 채워 넣으려던 스프라이트 시트 데이터는 실제 게임 에셋과 다르므로, 프롬프트대로 제가 직접 넣을 수 있게 비워두도록 수정했습니다.

## 적용 결과
- **잘 된 것:**
    - 게임 HUD(Lives, Score, Combo)의 가독성이 폰트와 그림자 효과 덕분에 훨씬 좋아졌습니다.
    - 배경음악이 게임 시작과 동시에 루프 재생되어 게임의 몰입감이 향상되었습니다.
    - 보스 패턴 로직과 충돌 판정이 의도대로 유지되었습니다.
- **어려웠던 것:**
    - UI를 수정하면서 기존의 복잡한 게임 시스템(보스 페이즈 전환, 탄막 생성 등)과 꼬이지 않게 병합하는 과정이 다소 까다로웠습니다.
