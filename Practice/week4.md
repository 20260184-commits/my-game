# 4주차 실습 정리

## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰

## 관찰 내용

### 원형  vs AABB
- 원형은 모서리 부분이 원형처럼 되어 있어 원형인 스프라이트에 적합하다
- AABB는 모서리가 각지게 되어있어 각이 있거나 사각형에 가까운 스프라이트에 사용할 것이다

### AABB vs OBB
- AABB는 회전과 상관없이 사각형에 크기가 일정하게 유지 되어서 회전하는 스프라이트에 적용시키기에는 적절하지 않다 대신 OBB에 비해 저비용이라서 회전할 일이 없으면 AABB를 쓰는 편이 좋다
- OBB는 스프라이트와 같이 회전하기 때문에 회전하는 스프라이트에 적절하다 대신 AABB에 비해 높은 비용을 쓰기 때문에 회전할 일이 없으면 쓰지 않는 편이 좋다

## AI와의 대화에서 배운 것
- math.hypot(dx, dy): 코드 가독성이 좋고, 매우 큰 수나 작은 수 계산 시 오버플로우/언더플로우를 방지하는 안정성이 뛰어납니다.
- math.sqrt 직접 사용: hypot와 결과는 같으나 수치적 예외 처리를 직접 고려해야 합니다.
- 게임 성능 최적화: 게임 개발에서는 sqrt(제곱근) 연산 자체가 CPU 자원을 많이 소모하므로 피해야 합니다. 따라서 sqrt 대신 양변을 제곱한 거리^2 < 반지름^2 방식을 사용하는 것이 성능상 훨씬 효율적입니다.
- 각도 업데이트 (angle): 매 프레임마다 angle 변수 값을 증가시켜 회전 상태를 갱신합니다. (Z키 입력 시 속도 증가)
- 이미지 회전 (pygame.transform.rotate): angle 값을 기준으로 원본 이미지를 회전시킨 새로운 이미지를 생성합니다.
- 중심점 고정 (get_rect(center=...)): 회전 시 이미지 크기 변화로 인해 위치가 어긋나지 않도록, 중심 좌표를 기준으로 다시 고정합니다

## 내 게임에 적용한다면
- 주로 OBB 방식을 채택할 것 같습니다. 왜냐하면 게임에 입체적인 움직을 위해 회전을 주로 쓸거 같아서 OBB 방식을 주로 사용할 것 같습니다.
